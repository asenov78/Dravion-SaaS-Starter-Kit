# Security — Dravion SaaS Starter Kit

## Secrets Management

| Secret | Location | Notes |
|--------|----------|-------|
| `APP_KEY` | `.env` | Generated by installer, never committed |
| `DRAVION_LICENSE_KEY` | `.env` | Never hardcode in config files |
| `DRAVION_LICENSED_DOMAIN` | `.env` | Must match the domain making license requests |
| `GITHUB_TOKEN` | `.env` | Fine-grained PAT, `contents: read` scope only, scoped to this repo |
| DB credentials | `.env` | Never committed |
| SMTP credentials | `.env` or Settings DB | Settings UI stores in DB, not `.env` |

`.env` is in `.gitignore`. Never commit it.

---

## Authentication & Authorization

- Standard Laravel session auth (bcrypt passwords, CSRF protection on all forms).
- All admin routes require `auth` + `verified` middleware.
- **Double guard:** permission checked at route level (`can:` middleware) AND inside controller (`$this->authorize()` / `Gate::check()`).
- Spatie `laravel-permission` handles role/permission assignment. Permissions are per-route granular.
- Super-admin bypass: the `super-admin` role bypasses all permission checks (Spatie default behavior).

## File Uploads

### Avatar Upload
- MIME validated server-side before processing.
- **GD re-encode:** image is decoded and re-encoded via GD (`imagecreatefromstring` → resize → `imagejpeg`). This strips any embedded payloads (EXIF shellcode, polyglot files).
- Output stored in `storage/app/public/avatars/{user_id}.jpg`.
- Max size: 2MB enforced in validation.

### Logo Upload
- Same GD re-encode pipeline as avatars.
- Stored in `storage/app/public/` as `logo.{ext}`.

## Updater Security

- Install gated by `LicenseService::isValid()` — license check before any file operation.
- Update package downloaded to `storage/app/updates/` (not public-accessible).
- **Protected paths** never overwritten: `.env`, `storage/`, `vendor/`, `node_modules/`, `public/storage/`, `public/build/`.
- After extract: runs `php artisan migrate --force` then clears caches.
- GitHub token is optional for public repos; when used, must be fine-grained PAT with minimum scope.

---

## Install Wizard

- Route `/install` only exists before installation.
- After install, a lock file is written (`storage/app/installed`) and the route is blocked by middleware.
- Installer self-deletes its own route registration reference after locking.
- If lock file is accidentally deleted, re-running the installer will only re-run migrations (idempotent), not re-create admin or overwrite license.

## OWASP Quick Checklist

| Risk | Status |
|------|--------|
| SQL Injection | Eloquent ORM throughout, no raw queries with user input |
| XSS | Blade `{{ }}` auto-escapes, `{!! !!}` used only for trusted content |
| CSRF | Laravel CSRF middleware active on all POST/PATCH/DELETE routes |
| Broken Auth | Session auth, bcrypt, no remember-me token reuse |
| Insecure Upload | GD re-encode pipeline, MIME check, size limit |
| IDOR | Resource fetched by ID checked against ownership or permission |
| Security Misconfiguration | `.env` out of git, installer locks, debug off in prod |
| Sensitive Data Exposure | No secrets in logs, no stack traces in prod (`APP_DEBUG=false`) |

## Logging & Audit Trail

All admin actions logged via `spatie/laravel-activitylog`. Log entries include:
- `causer` (who did it)
- `subject` (what was changed)
- `properties.old` / `properties.new` (diff)
- Timestamp

Logs are **not auto-purged** — implement a scheduled `activity()->where('created_at', '<', now()->subDays(90))->delete()` for retention policy.
